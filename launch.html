A4Create the proofs storage bucket

Why: Kids upload proof photos. Bucket must be private (not public). One click in the dashboard.
Open Supabase → Storage ↗
Click New bucket → name: proofs → toggle Public bucket OFF (stays private) → Create bucket.

A5Rotate the JWT keys (security)

Why: You pasted the anon + service_role keys in chat earlier. Rotate both now and update .env.local with the new ones.
Open Supabase → API settings ↗
1. Click Reset service_role key → confirm → copy the new value → paste into .env.local as SUPABASE_SERVICE_ROLE_KEY
2. Click Reset anon key → confirm → copy → paste as NEXT_PUBLIC_SUPABASE_ANON_KEY
3. Save .env.local

Section B — Railway deploy

Push to Railway, configure env vars, get a public URL.

B1Log in to Railway

Opens a browser tab for OAuth — sign in with your existing Railway account (same one Holy Flow uses).
railway login

B2Create a new Railway project

Creates an empty Railway project named family-accountability and links this folder to it.
cd C:/Users/User/family-accountability
railway init --name family-accountability
If it asks "Select an environment" — choose production (or accept the default).

B3Push env vars to Railway

Reads every variable from your local .env.local and pushes each to the Railway project. ~25 vars.
node scripts/push-env-to-railway.mjs

B4Deploy

Builds and deploys the Next.js app. ~2–3 minutes first time.
railway up

B5Get a public URL

Creates a free *.up.railway.app domain. Copy the URL it prints.
railway domain
